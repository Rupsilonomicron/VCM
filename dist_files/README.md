# VCM — VoiceChatMover v1.6.0

大会やイベントで、メインVC ⇄ チーム別VC の分散・集合をブラウザGUIから素早く行うための
Discord bot です。**Python のインストールは不要**（実行環境同梱）。

VOICEVOX がインストールされた PC では、VC のチャット**読み上げ機能**も使えます
（VOICEVOX が無くても VC 編成機能はすべて使えます）。

## 使い方

### 1. Discord Bot を作成（初回のみ）

1. https://discord.com/developers/applications → **New Application** → 名前を付けて作成
2. 左メニュー **Bot** ページで:
   - **Reset Token** → 表示されたトークンをコピー（あとで使います）
   - **Privileged Gateway Intents** の **SERVER MEMBERS INTENT** と
     **MESSAGE CONTENT INTENT** を両方 **ON**
3. 左メニュー **OAuth2 → URL Generator** で:
   - Scopes: `bot` と `applications.commands`
   - Bot Permissions: `View Channels` / `Send Messages` / `Connect` / `Speak` / `Move Members`
   - 生成された URL を開き、使いたいサーバーに bot を招待

### 2. 起動

`start_bot.bat` をダブルクリック。ブラウザで http://127.0.0.1:8765 が開きます。

### 3. トークンを設定（初回のみ）

初回はトークン設定画面が自動で開くので、手順1でコピーした Bot トークンを貼り付けて
**「保存して接続」**。トークンはこの PC の `config.json` にのみ保存され、
次回からは自動で接続されます。

変更・削除はヘッダの **「⚙ トークン設定」** からいつでもできます。

> **注意**: Bot トークンは絶対に他人に教えないでください。
> このフォルダを誰かに渡すときは `config.json` を削除してから渡してください。

## できること

- サーバー内のVC一覧・接続ユーザー（名前＆アイコン）・チームをリアルタイム表示
- 複数サーバー対応（ヘッダのプルダウンで切替）
- メンバーカードをドラッグ＆ドロップ:
  - 別のVC列へ → そのユーザーを実際にVC移動
  - チーム列へ → チームに割り当て（1人1チーム）
- **範囲選択**: パネルの余白からドラッグして枠で囲むと複数人を一括選択でき、
  まとめてドラッグ移動できます（Ctrl+ドラッグで追加選択、余白クリックで選択解除）
- チームヘッダをVC列へドラッグ → チーム全員を一括移動
- メインVCを設定して **集合**（チーム所属者全員をメインVCへ）／
  **散開**（各チームを「散開先VC」へチームごと移動）
  - 散開先は、チーム見出しをVCへドラッグして移動したときに記録されます
  - どのVCがどのチームの散開先かは、VC一覧の ⛺ バッジで確認できます
- **🔀 シャッフル**: チーム所属メンバーをランダムかつ均等に振り分け直し。
  チーム内のメンバーカードにマウスを乗せて **📌** を押すとピン留めでき、
  ピン留めした人（リーダーなど）はシャッフルしてもそのチームから動きません
- **✋ 参加希望を募る**: メインVCのチャットに「参加希望」ボタンを設置。
  メインVCに接続中の人がボタンを押すと、本人にだけ見えるチーム選択が出て、
  好きなチームに参加できます（選択後または60秒放置でメッセージは自動で消えます）
- チーム編成の **プリセット保存／呼び出し**
- 新しいバージョンが公開されるとヘッダーに通知が表示され、クリック →
  「アップデートして再起動」で自動アップデート（トークンなどの設定は引き継がれます）
- ブラウザのタブを閉じるとアプリも自動終了

## 読み上げ機能（VOICEVOX が必要）

PC に [VOICEVOX](https://voicevox.hiroshiba.jp/)（無料）がインストールされていれば、
起動時に自動で検出・接続され、GUI に読み上げバーが表示されます。
未インストールでも VC 編成機能はそのまま使えます（あとから入れて「再検出」でもOK）。

- **メインVC を設定すると bot が自動でメインVC に入る**
  （メインVC を変更すると bot も自動で移動、未設定に戻すと退出）
- **入室中 VC の内蔵テキストチャット**に書き込まれたメッセージを読み上げ
- 各ユーザーは **`/voice` コマンド**で自分の声（キャラクター・スタイル）を変更可能
- **📖 辞書**: 単語の読み方を登録（サーバーごとに保存。例: 「VCM」→「ぶいしーえむ」）
- **⏭ スキップ / キュー消去**: 長い読み上げを打ち切り・詰まり解消
- URL は「URL」、カスタム絵文字は名前だけ、長文は 120 文字で省略して読みます

> 本ソフトウェアの読み上げ機能は [VOICEVOX](https://voicevox.hiroshiba.jp/) を利用しています。
> VOICEVOX のキャラクターボイスを使用する際は、[VOICEVOX 利用規約](https://voicevox.hiroshiba.jp/term/)
> および各キャラクターごとの利用規約に従ってください。

## 制約（Discord 仕様）

- 移動できるのは **すでにVCに接続しているユーザーのみ**。VCにいない人は呼び出せません。
- bot に `Move Members` 権限と対象VCへの閲覧/接続権が必要です。

## うまく動かないとき

- **「トークン無効」と表示される** → トークンのコピーミス、または Developer Portal で
  トークンを再生成した可能性があります。Reset Token で新しいトークンを取得して設定し直してください。
- **「特権インテントが無効です」と表示される** → Developer Portal の Bot ページで
  SERVER MEMBERS INTENT と MESSAGE CONTENT INTENT を両方 ON にしてから、
  トークン設定画面で再度「保存して接続」。
- **サーバーが表示されない** → bot が対象サーバーに招待されているか確認してください。
- **移動できない** → bot の権限（Move Members）と、対象VCの閲覧/接続権を確認してください。
- **「VOICEVOX 未検出」と表示される** → VOICEVOX をインストールして「再検出」を押してください。
  特殊な場所にインストールしている場合は `config.json` に
  `"voicevox_path": "インストール先"` を追記。
- **読み上げの声が変わらない・入室できない** → bot の権限（Connect / Speak）と、
  招待 URL に `applications.commands` スコープが含まれているかを確認してください。
- **ポートが使用中と出る** → `config.json`（無ければメモ帳で新規作成）に
  `{ "port": 8766 }` のようにポート番号を書いてください。すでにトークン設定済みの場合は
  `"discord_token"` の行を消さないよう注意。
  なお `"host"` は変更しないでください（変更すると同じネットワークの誰でも
  VCM を操作できるようになってしまいます）。

## フォルダ構成

```
VCM/
  start_bot.bat    ← これをダブルクリックで起動
  python/          同梱の Python 実行環境（触らなくてOK）
  vcm/  web/       アプリ本体
  config.json      Bot トークンなどの設定（初回設定時に自動生成・共有しないこと）
  presets.json     チーム編成プリセット（自動生成）
  tts.json         読み上げの声設定・辞書（自動生成）
```
